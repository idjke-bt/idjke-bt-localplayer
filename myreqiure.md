目前项目基本功能正常，但界面展示需要向 Emby 的视觉效果靠拢，请按以下四个要求优化前端和部分后端逻辑。

## 1. 整体风格向 Emby 靠拢（纯文字描述，无参考图）
请将前端 UI 调整为更接近 Emby 的视觉风格，包括但不限于：
- 页面背景为接近纯黑的深色（#0a0a0a 或 #111），不要用灰色渐变。
- 顶部固定导航栏，左侧是“媒体库”标题，右侧是“设置”图标按钮，类似 Emby 的顶栏。
- 海报卡片具有明显的圆角（8-12px）、微妙的阴影和悬停时轻微上浮 + 放大的动效，卡片背景色与页面背景有区分（如 #1a1a1a），悬停时出现半透明播放按钮图标（使用 CSS 三角形或 Unicode ▶ 符）。
- 海报墙滚动时顶栏不消失，底部也没有多余留白。
- 字体优先使用系统无衬线字体（Segoe UI, Roboto, Helvetica 等），字号层级清晰。
- 详情页布局：左侧大尺寸海报（固定宽约 300px），右侧信息区，背景使用 fanart（见第 4 点）。

## 2. 海报墙下方文字左右居中
目前海报墙下方显示的电影名称和年份是左对齐或跟随网格对齐，请修改为：
- 每张海报卡片下方的标题文字（电影名）和年份文字水平居中对齐。
- 标题在上，年份在下，两行文字均居中。
- 标题过长时自动省略号，但保留完整标题在鼠标悬停时的 tooltip 属性里（HTML title 属性即可）。

## 3. 详情页增加演职人员等信息（利用 nfo 数据）
当前详情页只展示了标题、简介、评分，内容较单调。标准 Kodi nfo 中包含以下常见字段，请解析并在详情页展示：
- `<director>`：导演
- `<credits>` 或 `<writer>`：编剧
- `<actor>` 列表（含 name 和 role 属性）：演员表（至少显示前 5 位）
- `<genre>`：类型（已解析但可能未展示）
- `<runtime>`：时长
请将这些信息在详情页右侧信息区结构化显示，例如：
导演: xxx
编剧: xxx
主演: xxx, xxx, xxx ...
类型: 动作 / 科幻
时长: 148 分钟
简介: (已存在)

如果 nfo 中某字段不存在，则该行不显示，保证界面简洁。
同时更新 scanner.py 中的 nfo 解析函数和 database.py 的字段（如需要添加字段存储导演、演员等）。

## 4. 将 fanart.jpg 用作详情页背景
目前每个电影文件夹中的 fanart.jpg 未被使用。请修改：
- 在详情页中，将 fanart.jpg 作为背景图显示在详情页的整个背景区域，而不是单纯的白/黑色背景。
- 效果要求：背景图覆盖整个详情页，但通过半透明深色遮罩（约 0.6-0.7 透明度黑色）叠加，使得前景文字清晰可读，类似 Emby 的详情页背景效果。
- 如果该电影没有 fanart.jpg，则回退为纯色背景（与主页一致）。
- 确保背景图通过 Flask 的静态文件服务提供，与海报图片类似（如 `/fanart/<movie_id>` 接口）。
